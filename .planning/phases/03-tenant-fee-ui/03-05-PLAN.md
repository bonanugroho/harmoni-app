| Boundary | Description |
|----------|-------------|
| FeeForm → mutation hook | Fee data crosses from user input to TanStack Query mutation. Client-side validation runs before submission. |
| FeeList → onDeleteFee | Delete intent flows through ConfirmDialog before mutation execution. |
| TenantDetailPage → useFees/useTenant | All data fetching via TQ hooks through service layer. No direct API calls. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-03-14 | Tampering | FeeForm validation | mitigate | Amount validated: > 0, <= monthlyFee (client-side). Effective date validated: not in past. Server also validates per Phase 2 backend rules. |
| T-03-15 | Denial of Service | Excessive mutation calls | accept | No rate limiting on frontend. Backend should handle rate limiting if needed. MVP scope — low risk for neighborhood-scale app. |
| T-03-16 | Tampering | ConfirmDialog delete flow | mitigate | Two-step confirmation prevents accidental deletion. Optimistic removal not used — mutation success drives UI update via cache invalidation. |
</threat_model>
