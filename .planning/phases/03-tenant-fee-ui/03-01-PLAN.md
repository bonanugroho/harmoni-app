    - Interface `ApiError { error: string; code?: string }`
    - `async function request<T>(url: string, options?: RequestInit): Promise<T>` that:
      - Prepends `API_BASE_URL` to `url`
      - Merges `credentials: 'include'` into options
      - Merges `headers: { 'Content-Type': 'application/json' }` unless options override it
      - On `response.status === 204`: returns `undefined as T`
      - Detects JSON via `response.headers.get('content-type')?.includes('application/json')`
      - For JSON responses: calls `response.json()`, if `!response.ok` throws `new Error(data.error || 'An unexpected error occurred')`, else returns `data as T`
      - For non-JSON responses: if `!response.ok` throws `new Error('Connection lost. Check your internet and try again.')`, else returns `response.text() as T`
    - Export type `RequestOptions = RequestInit` for consumers that need to type options

    - Test 4: throws Error with API error message on 400+ JSON response
    - Test 5: throws generic error on 400+ non-JSON response
    - Test 6: returns parsed JSON for successful 200 responses
    - Test 7: merges custom options (method, body) correctly into request

    Do NOT use fenced code blocks in the action. Configure the test to use `beforeEach` with `vi.clearAllMocks()`. Use `vi.spyOn(globalThis, 'fetch')` for mocking.
