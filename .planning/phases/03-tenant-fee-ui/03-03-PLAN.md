<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Hooks → TanStack Query cache | Query cache stores tenant/fee data in memory. No persistence. |
| ProtectedRoute → API | /auth/me call crosses to backend for PASETO validation. |
| AuthContext → children | User object passed via React context to child components (sidebar visibility). |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-03-07 | Information Disclosure | AuthContext user object | accept | User object contains role/territory needed for UI decisions. No sensitive data (no password, no token). |
| T-03-08 | Elevation of Privilege | ProtectedRoute role check | accept | Client-side role check is UX convenience only. Backend Casbin enforcement prevents unauthorized access regardless of frontend state. |
| T-03-09 | Tampering | Query cache invalidation | mitigate | Mutations always call `invalidateQueries` in `onSuccess`. This ensures stale data is never shown after mutations. |
| T-03-10 | Denial of Service | excessive query refetch | mitigate | `staleTime: 30000` in QueryClient default prevents refetch on every keystroke. Auth query uses `staleTime: 300000` (5 min) since session rarely changes. |
</threat_model>
