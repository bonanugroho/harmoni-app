    **Step B — Create migration 006 (tenants table):**
    Write `006_create_tenants_table.up.sql` with:
    - `CREATE TABLE IF NOT EXISTS tenants`
    - `id UUID PRIMARY KEY DEFAULT uuidv7()`
    - `block VARCHAR(10) NOT NULL`
    - `unit_number VARCHAR(10) NOT NULL`
    - `occupancy VARCHAR(20) NOT NULL` with `CHECK (occupancy IN ('occupied', 'vacant'))`
    - `monthly_fee NUMERIC(12,2) NOT NULL` (NUMERIC not float — financial precision per D-08)
    - `territory_id VARCHAR(50) NOT NULL REFERENCES territories(id)` per Phase 1 territory FK pattern
    - `created_at TIMESTAMP DEFAULT NOW()`
    - `updated_at TIMESTAMP DEFAULT NOW()`
    - `UNIQUE(block, unit_number, territory_id)` per D-07
    - Index on `territory_id`

    Write `006_create_tenants_table.down.sql` as `DROP TABLE IF EXISTS tenants CASCADE;`

    **Step C — Create migration 007 (mandatory_fees table):**
    Write `007_create_mandatory_fees_table.up.sql` with:
    - `CREATE TABLE IF NOT EXISTS mandatory_fees`
    - `id UUID PRIMARY KEY DEFAULT uuidv7()`
    - `tenant_id UUID NOT NULL REFERENCES tenants(id) ON DELETE CASCADE`
    - `amount NUMERIC(12,2) NOT NULL CHECK (amount >= 0)` per D-08
    - `description VARCHAR(255) NOT NULL`
    - `effective_date DATE NOT NULL` (DATE not TIMESTAMP — fee applicability is day-level)
    - `paid_at TIMESTAMP` (nullable — NULL means not yet paid)
    - `created_at TIMESTAMP DEFAULT NOW()`
    - Index on `tenant_id`

    Write `007_create_mandatory_fees_table.down.sql` as `DROP TABLE IF EXISTS mandatory_fees CASCADE;`

    **Step D — Create migration 008 (voluntary_fees table):**
    Write `008_create_voluntary_fees_table.up.sql` with:
    - Same structure as mandatory_fees except table name is `voluntary_fees`
    - `CREATE TABLE IF NOT EXISTS voluntary_fees`
    - All columns identical to mandatory_fees (id, tenant_id FK, amount, description, effective_date, paid_at, created_at)
    - No UNIQUE constraint per tenant (multiple voluntary entries allowed per D-10)

    Write `008_create_voluntary_fees_table.down.sql` as `DROP TABLE IF EXISTS voluntary_fees CASCADE;`

    **Step E — Create migration 009 (user_tenants junction table):**
    Write `009_create_user_tenants_table.up.sql` with:
    - `CREATE TABLE IF NOT EXISTS user_tenants`
    - `user_id UUID NOT NULL REFERENCES users(id) ON DELETE CASCADE`
    - `tenant_id UUID NOT NULL REFERENCES tenants(id) ON DELETE CASCADE`
    - `created_at TIMESTAMP DEFAULT NOW()`
    - `PRIMARY KEY (user_id, tenant_id)` — composite PK per D-06
    - Indexes on both user_id and tenant_id for reverse lookup

    Write `009_create_user_tenants_table.down.sql` as `DROP TABLE IF EXISTS user_tenants CASCADE;`

    **Step F — Create migration 010 (seed sample data):**
    Write `010_seed_sample_tenants.up.sql` with:
    - INSERT 2 sample tenants: one in rt-01 (Block A, Unit 01, occupied, 500000 monthly fee), one in rt-02 (Block B, Unit 01, vacant, 300000 monthly fee)
    - INSERT mandatory fees for each tenant (Security fee and Waste fee at various amounts)
    - INSERT a voluntary contribution for the occupied tenant
    - INSERT user_tenants linking the sample resident to their tenant
    - Use UUID literals for reproducibility (e.g., `'00000000-0000-0000-0000-000000000001'` style)

    Write `010_seed_sample_tenants.down.sql` as:
    ```
    DELETE FROM user_tenants;
    DELETE FROM voluntary_fees;
    DELETE FROM mandatory_fees;
    DELETE FROM tenants;
    ```
