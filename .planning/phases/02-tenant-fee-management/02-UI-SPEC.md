# Phase 2 — UI Design Contract

> Visual and interaction contract for Tenant & Fee Management frontend. Generated by gsd-ui-researcher, verified by gsd-ui-checker.

---

## Design System

| Property | Value |
|----------|-------|
| Tool | none (raw TailwindCSS 4 + custom CSS variables) |
| Preset | not applicable |
| Component library | none (hand-rolled components with Tailwind utility classes) |
| Icon library | Lucide React (`lucide-react`) |
| Font | System font stack: `-apple-system, BlinkMacSystemFont, "Segoe UI", Roboto, sans-serif` |

**Rationale:** Phase 1 established a working pattern without shadcn — custom CSS variables via TailwindCSS 4's `@theme` block, raw HTML elements with Tailwind utilities, and no component abstraction layer. Phase 2 continues this pattern for consistency. Adding `lucide-react` for essential action icons (Plus, Pencil, Trash2, ChevronLeft, Menu, X).

**Installation required:**
```bash
cd apps/web && npm install lucide-react
```

**New components to create for this phase (hand-rolled, not from registry):**
- `AppLayout` — Sidebar + header + content area shell
- `TenantCard` — Tenant summary card for list view
- `TenantForm` — Create/edit tenant form
- `FeeList` — List of fees per tenant (mandatory + voluntary sections)
- `FeeForm` — Create/edit fee form (inline or modal)
- `StatusBadge` — Occupancy / payment status indicator
- `ConfirmDialog` — Delete confirmation dialog
- `PageHeader` — Consistent page title + action button pattern
- `LoadingSkeleton` — Card/list loading placeholders
- `EmptyState` — Consistent empty state illustration + CTA pattern

**Design token extension (add to `index.css` `@theme`):**
```
--color-sidebar: #F3F4F6        /* Gray-100 — sidebar background */
```

## Spacing Scale

Declared values (must be multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon-to-text gaps, inline badge padding |
| sm | 8px | Card internal padding (compact), label-to-badge gap |
| md | 16px | Default element spacing (form field groups, card padding, button padding) |
| lg | 24px | Section padding (card section gaps, sidebar item padding) |
| xl | 32px | Layout gaps (page header to card grid, sidebar width on mobile drawer) |
| 2xl | 48px | Major section breaks (fee type sections, form to footer gap) |
| 3xl | 64px | Page-level spacing (top padding below sidebar header) |

**Exceptions for this phase:**
- Touch targets minimum 44px height (WCAG 2.5.5) — applied to all buttons, sidebar links, interactive cards
- Tenant card touch target: entire card is tappable (≥44px height minimum, ensured by content padding)
- Sidebar links: 44px minimum height with `py-2.5 px-4` (10px vertical padding + 24px line height = 44px)
- Confirm dialog buttons: minimum 44px × 96px (wide enough for "Delete"/"Cancel" labels)
- Icon-only buttons (sidebar toggle): 44px × 44px minimum touch target

## Typography

| Role | Size | Weight | Line Height | Usage |
|------|------|--------|-------------|-------|
| Body | 16px | 400 (regular) | 1.5 | Form labels, card body text, fee descriptions, table data |
| Small | 14px | 400 (regular) | 1.5 | Helper text, error messages, status badges, timestamps, secondary info |
| Label | 14px | 600 (semibold) | 1.5 | Form field labels, section subtitles, fee type headers |
| Heading | 24px | 600 (semibold) | 1.2 | Page titles ("Tenants", "Unit A-01 — Fees") |
| Display | 28px | 600 (semibold) | 1.2 | App name in sidebar header ("Harmoni") |

**Rationale:** Keeps the 2-weight (400, 600) system with exactly 4 sizes. Body 16px continues as the primary readable size. Small 14px covers labels, helper text, and badges (with weight distinguishing label from body). Heading 24px is sufficient for phase pages. Display 28px reserved for the sidebar brand.

**Currency amounts:** 16px semibold for price emphasis within cards, 24px semibold for total/summary figures.

## Interaction Patterns

### Form Submission (extends Phase 1 pattern)
- Client-side validation before submit
- Server errors display as top-of-form alert (`role="alert"`)
- Submit button shows spinner + "Saving..." text during API call
- Prevent double-submission (button disabled until response)
- On success: redirect to tenant list (for tenant create) or fee list (for fee create)
- On error: keep form state, show error message

### Sidebar Navigation
- Collapsible via hamburger icon on mobile / collapse button on desktop
- Active route highlighted with blue-100 background + left border indicator
- Navigation changes trigger route change (React Router `useNavigate`)
- Sidebar items show tooltip on collapsed state (desktop)
- Smooth CSS transition on expand/collapse (no JavaScript animation libs)

### Tenant Card Interaction
- Entire card is clickable (no dedicated "View" button)
- Hover state: subtle shadow increase (`shadow-sm` → `shadow-md`)
- Active/pressed state: slight scale down or background color change
- Touch feedback: `active:bg-gray-50` or `active:scale-[0.98]`

### Delete Confirmation (ConfirmDialog component)
- Triggered by clicking trash icon on fee card or "Delete Tenant" button
- Modal overlay with backdrop
- Dialog content: warning message + two buttons
- Cancel button (outline style) dismisses dialog
- Delete button (destructive red) performs deletion
- On successful delete: remove item from list (optimistic) OR show success message + redirect
- On failure: close dialog, show error toast/alert

### Loading States
| Page | Loading Pattern |
|------|----------------|
| Tenant list | 3 skeleton cards (rounded rectangles with pulse animation, `animate-pulse`) |
| Tenant detail / fee list | Skeleton info header + 3 skeleton fee rows |
| Form create | No skeleton (form fields rendered immediately) |
| Form edit | Skeleton form while fetching tenant data (then reveal fields) |
| After save | Button spinner + "Saving..." → redirect on success |
| After delete | Brief loading state → redirect or remove item |

### Search / Filter (Tenant List)
- Client-side filter input at top of tenant list
- Filters by: block name, unit number
- Debounced 300ms input
- No server-side search for MVP (client-side filter on already-loaded tenants)
- Clear search button when input has value

### Pull-to-Refresh (Mobile Tenant List)
- Mobile browsers' native pull-to-refresh behavior
- On refresh, re-fetch tenant list from API
- Show loading indicator at top during refresh

### Fee Status Toggle
- Unpaid → Paid: inline action (click unpaid badge opens date picker or marks as paid immediately)
- For MVP: mark as paid by editing the fee and setting payment date
- No "mark all paid" bulk action in this phase

| Breakpoint | Layout Behavior |
|------------|-----------------|
| < 640px (default mobile) | Sidebar hidden (hamburger toggle), full-width cards, single column grid, forms full-width, modals full-screen |

- No CSS animations or transitions that require GPU compositing (except `opacity` and `transform` for sidebar)
- No `backdrop-filter` or blur effects (sidebar backdrop uses solid gray-900 with opacity)
- Minimal box-shadows (single layer, low blur radius: `shadow-sm`)
- System font stack — no web font download
- Touch targets minimum 44px × 44px (WCAG 2.5.5)
- Sidebar drawer uses CSS `translateX` transition only (hardware-accelerated)
- No `position: sticky` on sidebar (uses fixed positioning — sufficient for low-end)

## Accessibility Requirements

| Requirement | Implementation |
|-------------|----------------|
| WCAG 2.1 AA contrast | All text meets 4.5:1 minimum contrast ratio on its background (status badges: colored bg + dark text meets ratio) |
| Keyboard navigation | All sidebar links tabbable, all form fields tabbable in logical order, Enter submits form, Escape closes dialog |
| Screen reader labels | All inputs have associated `<label>` elements, error messages use `aria-describedby`, sidebar toggle uses `aria-label`, dialog uses `role="dialog"` + `aria-modal` |
| Focus management | After form submission error, focus moves to first error field. After dialog close, focus returns to trigger button |
| Reduced motion | Respect `prefers-reduced-motion` — disable sidebar slide animation, disable card hover scale, disable skeleton pulse |
| Form errors | Inline errors use `role="alert"` for screen reader announcement |
| Skip link | "Skip to content" link as first focusable element on page (hidden until focused via keyboard) |
| Dialog | Focus trap inside dialog, close on Escape, close on backdrop click |
