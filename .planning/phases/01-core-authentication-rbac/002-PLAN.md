### Task 4: Auth Service (Business Logic)
<read_first>
- apps/api/internal/domain/service/auth_service.go (create)
- apps/api/internal/domain/repository/user_repository.go
- apps/api/internal/infrastructure/auth/paseto.go
- apps/api/internal/infrastructure/auth/password.go
</read_first>

<action>
Implement authentication service:
- Register(email, password, fullName, territoryID string) → (*User, error)
- Login(email, password string) → (*User, string, error)  // returns user + token
- ResetPasswordRequest(email string) → error
- ResetPassword(token, newPassword string) → error
- Registration: validate password, hash password, create user (PostgreSQL generates UUIDv7)
- Login: find user, compare password, generate PASETO token
- Reset: generate one-time token, store in password_reset_tokens table (PostgreSQL generates UUIDv7)
</action>

<acceptance_criteria>
- apps/api/internal/domain/service/auth_service.go implements all auth methods
- Register creates user with UUIDv7 ID (from PostgreSQL), hashed password, returns user without password_hash
- Login returns user + valid PASETO token for correct credentials
- Login returns error for incorrect email or password
- ResetPasswordRequest creates token record with UUIDv7 ID (from PostgreSQL) and 1-hour expiry
- ResetPassword validates token, updates password, marks token as used
- Used or expired reset token returns error
- All UUIDs are UUIDv7 (generated by PostgreSQL 18's native `uuidv7()` function)
- `go test ./internal/domain/service -v -run TestAuthService` passes
</acceptance_criteria>

<action>
Implement HTTP handlers for auth endpoints:
- POST /auth/register: email, password, full_name, territory_id → 201 + user
- POST /auth/login: email, password → 200 + httpOnly cookie
- POST /auth/reset: email → 200 + email sent
- POST /auth/reset/confirm: token, new_password → 200 + password updated
- Set httpOnly cookie: Name="paseto_token", HttpOnly=true, Secure=true, SameSite=Strict, MaxAge=3600
- Return consistent error format: {"error": "message", "code": "ERROR_CODE"}
</action>

<acceptance_criteria>

- POST /auth/register returns 400 for invalid email format or weak password
- POST /auth/register returns 409 for duplicate email
- POST /auth/login returns 200 with httpOnly cookie set
- POST /auth/login returns 401 for invalid credentials
- POST /auth/reset returns 200 even if email doesn't exist (prevent enumeration)
- POST /auth/reset/confirm returns 200 for valid token
- POST /auth/reset/confirm returns 400 for expired or used token
- Cookie attributes: HttpOnly=true, Secure=true, SameSite=Strict, Path="/"
- `go test ./internal/delivery/http -v -run TestAuthHandler` passes
</acceptance_criteria>

## Verification

1. **Registration Flow:**
   ```bash
   curl -X POST http://localhost:8080/auth/register \
     -H "Content-Type: application/json" \
     -d '{"email":"test@example.com","password":"SecurePass123!","full_name":"Test User","territory_id":"rt-01"}'
   ```
   Expected: 201 + {"id": "...", "email": "test@example.com", ...}

2. **Login Flow:**
   ```bash
   curl -v -X POST http://localhost:8080/auth/login \
     -H "Content-Type: application/json" \
     -d '{"email":"test@example.com","password":"SecurePass123!"}'
   ```
   Expected: 200 + Set-Cookie header with paseto_token

3. **Password Reset Flow:**
   ```bash
   curl -X POST http://localhost:8080/auth/reset \
     -H "Content-Type: application/json" \
     -d '{"email":"test@example.com"}'
   ```
   Expected: 200 + email sent (check mock in tests)

4. **Token Validation:**
   ```bash
   # Extract cookie from login response, use in subsequent request
   curl -b "paseto_token=<token>" http://localhost:8080/protected
   ```
   Expected: 200 if token valid, 401 if invalid/expired
