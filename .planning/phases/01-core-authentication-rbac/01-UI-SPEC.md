# Phase 1 — UI Design Contract

> Visual and interaction contract for frontend phases. Generated by gsd-ui-researcher, verified by gsd-ui-checker.

---

## Design System

| Property | Value |
|----------|-------|
| Tool | shadcn (recommended — not yet initialized) |
| Preset | Default (neutral gray, CSS variables) — configure at `ui.shadcn.com/create` |
| Component library | Radix UI primitives (via shadcn) |
| Icon library | Lucide React (shadcn default) |
| Font | Inter (system fallback: -apple-system, BlinkMacSystemFont, "Segoe UI", sans-serif) |

**Rationale:** shadcn provides accessible, mobile-first components with Radix UI primitives — essential for low-end mobile browser compatibility. Inter is optimized for screen readability at small sizes.

**Initialization required before implementation:**
```bash
cd apps/web
npx shadcn@latest init
```

**Components needed for this phase (shadcn official registry):**
- `button` — CTAs, form actions
- `input` — Email, password, name fields
- `label` — Form field labels
- `card` — Auth form containers
- `alert` — Error/success messages
- `separator` — Visual dividers in forms

## Spacing Scale

Declared values (must be multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon-to-text gaps, inline padding within inputs |
| sm | 8px | Compact element spacing (label-to-input gap, checkbox-to-label) |
| md | 16px | Default element spacing (form field groups, button padding) |
| lg | 24px | Section padding (card internal padding, form section gaps) |
| xl | 32px | Layout gaps (page header to content, between auth cards) |
| 2xl | 48px | Major section breaks (form sections, logo-to-form gap) |
| 3xl | 64px | Page-level spacing (vertical centering offset on auth pages) |

Exceptions:
- Touch targets minimum 44px height (WCAG 2.5.5) — applied to all buttons and interactive inputs
- Password visibility toggle icon: 20px touch target minimum

## Typography

| Role | Size | Weight | Line Height | Usage |
|------|------|--------|-------------|-------|
| Body | 16px | 400 (regular) | 1.5 | Form labels, helper text, body copy |
| Small | 14px | 400 (regular) | 1.5 | Helper text, error messages, footnote |
| Heading | 24px | 600 (semibold) | 1.2 | Page titles ("Sign In", "Create Account") |
| Display | 28px | 600 (semibold) | 1.2 | App logo/brand on auth pages |

**Rationale:** 16px body is the mobile readability minimum. Only 2 weights (400, 600) reduces font file size for low-end devices. Line height 1.5 for body ensures readability on small screens.

## Color

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | `#FFFFFF` (white) / `#F9FAFB` (gray-50) | Page backgrounds, form surfaces |
| Secondary (30%) | `#F3F4F6` (gray-100) / `#E5E7EB` (gray-200) | Card backgrounds, input borders, separators |
| Accent (10%) | `#2563EB` (blue-600) | Primary CTA buttons, active input focus rings, links |
| Destructive | `#DC2626` (red-600) | Password validation errors, form submission errors, account lockout warnings |

**Dark mode:** Use CSS variables via shadcn's built-in dark mode support. Map to equivalent dark surface colors.

Accent reserved for:
- Primary action buttons ("Sign In", "Create Account", "Send Reset Link", "Set New Password")
- Active/focused input border rings
- "Forgot password?" link text
- Success state indicators (checkmarks, confirmation banners)

**Never use accent for:** Error states, disabled elements, secondary actions, decorative elements.

## Page Layouts

### Login Page
- Centered card layout (max-width 400px)
- App logo/brand at top
- Email input + Password input
- "Forgot password?" link below password field
- "Sign In" button (full-width)
- "Don't have an account? Create one" link at bottom
- Loading state: button shows spinner, disabled during submission

### Register Page
- Centered card layout (max-width 400px)
- App logo/brand at top
- Full Name input
- Email input
- Password input (with complexity hint below)
- Confirm Password input
- "Create Account" button (full-width)
- "Already have an account? Sign in" link at bottom
- Password visibility toggle on both password fields

### Reset Password — Request
- Centered card layout (max-width 400px)
- App logo/brand at top
- Email input
- "Send Reset Link" button (full-width)
- "Back to Sign In" link at bottom
- Success state replaces form with confirmation message

### Reset Password — Set New Password
- Centered card layout (max-width 400px)
- App logo/brand at top
- New Password input (with complexity hint)
- Confirm Password input
- "Update Password" button (full-width)
- Password visibility toggle on both fields

### Protected Route (post-auth redirect)
- After successful login, redirect to `/dashboard` (placeholder — Phase 4)
- If no role-specific dashboard exists yet, show a minimal welcome: "Welcome, {name}. Your dashboard is coming soon."

## Interaction Patterns

### Form Submission
- All forms use client-side validation before submission
- Server errors display as inline field errors or top-of-form alert
- Submit button disabled + loading spinner during API call
- Prevent double-submission (debounce 500ms)

### Password Visibility Toggle
- Eye icon (Lucide `Eye` / `EyeOff`) at right end of password input
- Toggle state persists only while input is focused
- Does not affect form validation state

### Session Management
- httpOnly cookies managed by server — no client-side token access
- On 401 response, redirect to `/login` with `?from={currentPath}` query param
- After login, redirect back to `from` path or `/dashboard`

### Loading States
- Form submission: button shows spinner + "Signing in..." / "Creating account..." / "Sending..."
- Page load: no skeleton needed for auth pages (forms are static)
- Redirect after auth: brief loading state with "Redirecting..." text

## Mobile-First Responsive Rules

| Breakpoint | Behavior |
|------------|----------|
| < 640px (default) | Full-width card with 16px horizontal padding, card fills viewport width minus padding |
| ≥ 640px (sm) | Card centered with max-width 400px, 24px padding |
| ≥ 768px (md) | Same as sm, with subtle card shadow |
| ≥ 1024px (lg) | Same as sm, with decorative background pattern (subtle geometric shapes) |

**Low-end mobile considerations:**
- No CSS animations or transitions that require GPU compositing
- No backdrop-filter or blur effects
- Minimal box-shadows (single layer, low blur radius)
- System font stack preferred over web font download (Inter as progressive enhancement)
- Touch targets minimum 44px × 44px

## Accessibility Requirements

| Requirement | Implementation |
|-------------|----------------|
| WCAG 2.1 AA contrast | All text meets 4.5:1 minimum contrast ratio on its background |
| Keyboard navigation | All form fields tabbable in logical order, Enter submits form |
| Screen reader labels | All inputs have associated `<label>` elements, error messages use `aria-describedby` |
| Focus management | After form submission error, focus moves to first error field |
| Reduced motion | Respect `prefers-reduced-motion` — disable any transitions |
| Form errors | Inline errors use `role="alert"` for screen reader announcement |
